# 投資部 シグナル検知スクリプト 使い方

## これは何か
指定した個別株を毎日チェックして、「買い時・売り時」のサインが出たら
`keiyou233501@gmail.com` にメールでお知らせするツールです。
**実際の注文は自動化していません。** 発注はメールを見てからご自身でSBI証券にて行ってください。

## 初回セットアップ（最初の1回だけ）

### 1. 必要なライブラリをインストール
コマンドプロンプトかPowerShellでこのフォルダに移動して実行します。

```
pip install -r requirements.txt
```

### 2. Gmailの「アプリパスワード」を発行する
メール送信のため、Googleアカウントの通常パスワードとは別に「アプリパスワード」という
16桁の専用パスワードが必要です（Gmailの仕様上、外部プログラムからのログインには専用パスワードが必要なため）。

1. https://myaccount.google.com/apppasswords を開く（2段階認証が有効になっている必要があります）
2. アプリ名を適当に入力（例: 投資部シグナル）して「作成」
3. 表示された16桁のパスワードをコピーしておく（スペースは詰めても詰めなくてもどちらでも可）

### 3. アプリパスワードを環境変数に設定する

**PowerShellで一時的に設定する場合**（そのウィンドウを閉じると消えます）:
```powershell
$env:GMAIL_APP_PASSWORD = "発行された16桁のパスワード"
```

**恒久的にWindowsに設定する場合**（一度設定すれば次回以降も使える）:
```powershell
[System.Environment]::SetEnvironmentVariable("GMAIL_APP_PASSWORD", "発行された16桁のパスワード", "User")
```
設定後は一度PowerShellを閉じて開き直してください。

## 実行方法
```
python signal_check.py
```
シグナルが出ればメールが届きます。出なければ何も届きませんが、
`../logs/YYYY-MM-DD.md` に毎回チェック結果が記録されます。

## 毎日自動実行したい場合（Windowsタスクスケジューラ）
1. 「タスクスケジューラ」をWindowsの検索から開く
2. 「基本タスクの作成」→ 名前を「投資部シグナルチェック」などにする
3. トリガー: 「毎日」を選び、時刻は日本株の取引終了後（15:30以降）がおすすめ
4. 操作: 「プログラムの開始」→ プログラム/スクリプトに `python`、引数に
   `"C:\Users\keiyo\Desktop\作業場\.company\投資部\scripts\signal_check.py"` を指定
5. 「開始（オプション）」にこのscriptsフォルダのパスを入れる

※ タスクスケジューラ経由だと環境変数 `GMAIL_APP_PASSWORD` が引き継がれないことがあるため、
うまく送信できない場合は手順3を「恒久的に設定する場合」の方法にしてください。

## ウォッチリスト・条件を変更したい場合
`config.py` を開いて編集してください。銘柄の追加・削除、損切り/利確ラインの調整などが可能です。

## 保有中の銘柄を登録する（売りシグナルを検知させるため）
買いシグナルのメールを見て実際にSBI証券で買ったら、`positions.json` に手動で追記してください。

```json
{
  "6857.T": {
    "entry_price": 12345,
    "entry_date": "2026-07-05",
    "shares": 1
  }
}
```
`shares`（購入株数）を入れておくと、ダッシュボードに円換算の損益も表示されます。省略しても%表示は動きます。
売却したらこのエントリーを削除してください。

**注意**: `positions.json` はGitには登録されません（`.gitignore`で除外済み）。理由は、
このリポジトリが後述のGitHub Pages自動公開のために公開設定になっており、実際の取得価格・
保有株数が誰でも見える状態になるのを防ぐためです。ローカルのファイルとしては引き続き
使えるので、手元で `dashboard.py` を実行してArtifactに公開する分にはこれまで通り動作します。

## GitHubでの自動更新について
`.github/workflows/update-dashboard.yml` により、毎営業日15:35(日本時間)に自動で
`dashboard.py` を実行し、結果をGitHub Pagesに公開します。ただし**この自動公開版は
保有銘柄の情報を一切含みません**（`PUBLIC_BUILD=1` という環境変数で強制的に「未保有」
として生成される）。買いシグナルのウォッチリストだけを見るための公開ページという位置づけです。
自分の保有銘柄の損益を確認したい場合は、今まで通り手元で `python dashboard.py` を実行し、
生成された `dashboard.html` をClaudeのArtifact機能で公開してください（このときは
`PUBLIC_BUILD` を設定しないので、`positions.json` の内容がそのまま反映されます）。
